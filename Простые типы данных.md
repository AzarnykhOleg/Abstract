# Строгая динамическая типизация Python

	Python является языком со строгой  динамической типизацией. Это означает что тип объекта изменить невозможно, он  строго задаётся при создании объекта. При этом переменные могут ссылаться на объекты разных типов. И это не вызывает ошибки.

# [Функция type(object)](zotero://open-pdf/library/items/J46R29EE?page=5&annotation=L7YCRIPX)
	- возвращает тип данных объекта или возвращает объект нового типа на основе переданных аргументов

```python
a=5  
print(type(a))  
a = "hello world"  
print(type(a))  
a = 42.0 * 3.141592 / 2.71828  
print(type(a))
```
>	Меняется класс объекта, на который ссылается переменная. 
>	Переменная динамически изменяется.

`type(name, bases, dict)` - возвращает объект нового типа:
>	`name` - название класса
>	`bases` - кортеж, определяющий базовый класс
>	`dict` - словарь, который представляет собой пространство имён, содержащее определения для тела класса

# Функция id(object)

 	- возвращает уникальный идентификатор объекта в памяти
Динамическая типизация:
```python
a=5  
print(type(a), id(a))  
a = "hello world"  
print(type(a), id(a))  
a = 42.0 * 3.141592 / 2.71828  
print(type(a), id(a)
```

	Каждый раз возвращается новый адрес объекта в памяти. Python не меняет класс  объекта. Он меняет переменную. Каждый раз она ссылается на новый объект. Такая  механика прописана в самом интерпретаторе.

# [Функция isinstance(object, classinfo)](zotero://open-pdf/library/items/J46R29EE?page=6&annotation=FSAK2R3D)

	Функция принимает на вход объект и класс и возвращает истину, если объект  является экземпляром прямого или косвенного подкласса.

```python
data = 42  
print(isinstance(data, int))
```

```python
data = True  
print(isinstance(data, int))
```

	Проверка может проходить сразу по нескольким классам. В этом случае  истина вернётся, если объект является экземпляром любого из переданных в кортеже классов.

```python
data = 3.14  
print(isinstance(data, (int, float, complex)))
```

***❗❗❗ Для проверки типа можно вызывать функцию type() и сравнить  значение с результатом. Но документация к языку не рекомендует так делать.  Для получения информации о типе объекта правильно использовать функцию  isinstance(). ***

# [Оператор is](zotero://open-pdf/library/items/J46R29EE?page=7&annotation=5FACKDLN)
	- сравнивает не значения, а сами объекты

***❗❗❗ "is" НЕ "==" ***

```python
num = 2 + 2 * 2  
digit = 36 / 6  
print(num == digit)  
print(num is digit)
```

	По сути оператор is занимается сравнением "айдишников" объектов, т.е. проверяет  лежит ли объекты в одном месте в памяти компьютера.

# [Изменяемые и неизменяемые типы](zotero://open-pdf/library/items/J46R29EE?page=7&annotation=D8E3T3YQ)

	Изменяемые (mutable) - объекты, содержимое которых можно исправлять после их создания
	Неизменяемые (immutable) - объекты, содержимое которых нельзя изменить после их создания


| Неизменяемые                          | Изменяемые                          |
| ------------------------------------- | ----------------------------------- |
| None                                  |                                     |
| Числа: int, bool, float, complex      |                                     |
| Последовательности: str, tuple, bytes | Последовательности: list, bytearray |
| Множества: frosenset                  | Множества: set                      |
|                                       | Отображения: dict                   |

# [Хэш hash(object) как проверка на неизменяемость](zotero://open-pdf/library/items/J46R29EE?page=9&annotation=NW89L4I8)

	Вариант убедиться в изменяемости или неизменяемости — проверка  на хеширование.

	В Python для получения хеша используется функция `hash()`. 
	Если объект  является неизменяемым, функция возвращает число — хеш-сумму.
	  Изменяемые объекты хешировать нельзя исходя из самого определения  хеш-функции.

```python
x = 42  
y = "text"  
z = 3.1415  
print(hash(x), hash(y), hash(z)) 
 
my_list = [x, y, z]  
print(hash(my_list))
```
