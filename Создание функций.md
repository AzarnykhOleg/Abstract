# [Создание своих функций](zotero://open-pdf/library/items/RX6G9YZB?page=5&annotation=HR5I3HEZ)

# [Аргументы функции](zotero://open-pdf/library/items/RX6G9YZB?page=7&annotation=6YEXAQUH)

	Аргументы - значения, передаваемые в функцию при вызове. Параметры - переменные в определении функции, которые принимают эти значения. В Python аргументы передаются по ссылке (передается ссылка на объект).

```python
def greet(name, age):  # name и age - параметры
    print(f"Привет, {name}, тебе {age} лет")
greet("Анна", 25)  # "Анна" и 25 - аргументы
```

	Неизменяемые аргументы - при изменении внутри функции остаются прежними вне функции.

	Изменяемые аргументы - при изменении внутри функции изменяются и за её пределами.

> Неизменяемый - число
```python

def add_one(x):
    x = x + 1  # создается новый объект
    return x

num = 5
add_one(num)
print(num)  # 5 (не изменилось)
```

> Изменяемый - список
```python

def append_one(lst):
    lst.append(1)  # изменяем объект по ссылке
my_list = [0]

append_one(my_list)
print(my_list)  # [0, 1] (изменилось)
```

# [Возврат значения](zotero://open-pdf/library/items/RX6G9YZB?page=9&annotation=AYNI45KQ)

	Оператор return прекращает выполнение функции и возвращает значение в точку вызова. Функция может возвращать любой объект: число, строку, список, словарь, другую функцию или кортеж значений.

```python
def square(x):
    return x ** 2

def multiple_returns():
    return 1, 2, 3  # возвращает кортеж (1, 2, 3)
a, b, c = multiple_returns()  # распаковка
```

# [Неявный return](zotero://open-pdf/library/items/RX6G9YZB?page=10&annotation=H79JAD7V)

	Если функция завершается без явного оператора return, или return указан без значения, функция неявно возвращает None. Это полезно для функций-процедур, которые выполняют действия, но не вычисляют результат.

```python
def print_message(msg):
    print(msg)
    # неявный return None
result = print_message("Hello")
print(result)  # None
```

# [Значения по умолчанию](zotero://open-pdf/library/items/RX6G9YZB?page=11&annotation=SPXS54BW)

	Параметрам можно присваивать значения по умолчанию через знак =. Если при вызове аргумент не передан, используется значение по умолчанию. Параметры со значениями по умолчанию должны следовать после обязательных параметров.

```python
def power(base, exponent=2):
    return base ** exponent

print(power(3))      # 9 (3^2)
print(power(2, 3))   # 8 (2^3)
```

# [Изменяемый объект как значение  по умолчанию](zotero://open-pdf/library/items/RX6G9YZB?page=11&annotation=R4VXP5JM)

	Использование изменяемого объекта (список, словарь) в качестве значения по умолчанию - опасная ловушка. Значение по умолчанию вычисляется один раз при определении функции, а не при каждом вызове, поэтому один и тот же объект используется во всех вызовах.

> НЕПРАВИЛЬНО:
```python

def bad_append(item, my_list=[]):
    my_list.append(item)
    return my_list


print(bad_append(1))  # [1]
print(bad_append(2))  # [1, 2] - ой, список сохранился!
```

> ПРАВИЛЬНО:
```python
def good_append(item, my_list=None):
    if my_list is None:
        my_list = []
    my_list.append(item)
    return my_list
    
    
print(good_append(1))
```

# [Позиционные и ключевые параметры](zotero://open-pdf/library/items/RX6G9YZB?page=12&annotation=R2QHNHTV)

	Позиционные аргументы передаются по порядку следования параметров. 
	
	Ключевые (именованные) аргументы передаются с указанием имени параметра=значение, порядок не важен. 
	
	Сначала должны идти позиционные, потом ключевые.

```python
def person(name, age, city="Москва"):
    print(f"{name}, {age} лет, {city}")


person("Иван", 30)           # позиционные
person(age=25, name="Мария") # ключевые
person("Петр", city="Казань", age=40) # смешанные
```

	Косая черта / и звёздочка * разделяют позиционные и ключевые параметры:

```python
def func(позиционные_параметры, /, позиционные_или_ключевые_параметры, *, ключевые_параметры):
	pass
```

```python
def func(arg, /)    # только позиционные
def func(arg)       # по ключу и по позиции
def func(*, arg)    # только по ключу
```

# [Параметры args и kwargs](zotero://open-pdf/library/items/RX6G9YZB?page=14&annotation=3IFDWJMI)

> *args собирает избыточные позиционные аргументы в кортеж (tuple). 

> **kwargs собирает избыточные именованные аргументы в словарь (dict). Позволяют создавать функции с переменным числом аргументов.


```python
def sum_all(*args):  # args - кортеж
    return sum(args)


print(sum_all(1, 2, 3, 4))  # 10
```

```python
def print_info(**kwargs):  # kwargs - словарь
    for key, value in kwargs.items():
        print(f"{key}: {value}")


print_info(name="Иван", age=30)
```

```python
# Комбинация
def universal(pos, *args, key="default", **kwargs):
    print(f"pos: {pos}, args: {args}, key: {key}, kwargs: {kwargs}")


universal(1, 2, 3, key="custom", extra="data")
```

# [Области видимости: global и nonlocal](zotero://open-pdf/library/items/RX6G9YZB?page=16&annotation=6DWI8ULR)

> Правило LEGB: Local (локальная) -> Enclosing (объемлющая) -> Global (глобальная) -> Built-in (встроенная). Для чтения переменных Python ищет их в этих областях слева направо.

> global объявляет, что переменная из глобальной области должна использоваться и изменяться внутри функции.

> nonlocal используется во вложенных функциях для указания, что переменная принадлежит объемлющей (родительской) функции, но не глобальной области.


```python
counter = 0
def global_example():
    global counter
    counter += 1

print(counter)



def outer():
    x = 10
    def inner():
        nonlocal x  # используем x из outer, а не создаем новую
        x += 5
        return x
    return inner()

print(outer())  # 15
```

# [Доступ к константам](zotero://open-pdf/library/items/RX6G9YZB?page=18&annotation=GD4NQHS8)

	Константы (переменные, которые не предполагается изменять) обычно определяются в глобальной области модуля заглавными буквами (соглашение PEP 8). Функции могут читать глобальные константы без использования global, так как они не изменяют их.

```python
PI = 3.14159
MAX_CONNECTIONS = 100

def circle_area(radius):
    # читаем глобальную константу без global
    return PI * radius ** 2
```

# [Анонимная функция lambda](zotero://open-pdf/library/items/RX6G9YZB?page=18&annotation=FME89MPF)

	lambda - выражение для создания маленьких безымянных функций с одним выражением в теле. Используется там, где нужна функция на короткий срок (параметр key в sort(), map(), filter()). Не может содержать statements (return, pass, циклы), только выражение.

```python
# Обычная функция
def add(x, y):
    return x + y
# Эквивалентная lambda
add = lambda x, y: x + y
# Использование без присваивания
numbers = [1, -2, 3, -4, 5]
positive = list(filter(lambda x: x > 0, numbers))
sorted_by_abs = sorted(numbers, key=lambda x: abs(x))
```

# [Документирование кода функций](zotero://open-pdf/library/items/RX6G9YZB?page=19&annotation=U25VTWAR)

	Докстринг (docstring) - строка документации, размещаемая сразу после заголовка функции в тройных кавычках. Доступен через атрибут __doc__ или функцию help(). Описывает назначение, параметры и возвращаемое значение.

```python
def calculate_bmi(weight, height):
    """
    Рассчитывает индекс массы тела (BMI).
    
    Args:
        weight (float): Вес в килограммах
        height (float): Рост в метрах
        
    Returns:
        float: Значение BMI
    """
    return weight / (height ** 2)
print(help(calculate_bmi))
```
