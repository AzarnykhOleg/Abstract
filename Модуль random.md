# import
([4](zotero://open-pdf/library/items/NW29V7HX?page=4&annotation=L2ZXNTAL))

***❗❗❗ PEP-8! Строки импорта рекомендуется писать в самом начале файла,  оставляя 1-2 пустые строки после него. ***

```python
import sys  print(sys)  print(sys.builtin_module_names)  print(*sys.path, sep=&#39;&#92;n&#39;)
```

	Механизм  импорта выполняет поиск указанного имени

	в  нескольких местах. В первую очередь проверяется список встроенных модулей,  который хранится в sys.builtin_module_names. Если имя модуля не найдено,  проверка ведётся в каталогах файловой системы, которые перечислены в sys.path.

> Переменная sys.path
([5](zotero://open-pdf/library/items/NW29V7HX?page=5&annotation=UYQWEIK7))

	Содержимое переменной sys.path формируется динамически. В качестве первого  адреса указывается путь до основного файла.

	Далее в sys.path перечислены пути из PYTHONPATH и пути, указанные при  установке Python и создании виртуального окружения.

***❗❗❗ Если создать собственный файл с именем аналогичным имени  модуля, Python импортирует ваш файл, а не модуль. Строго не рекомендуется  использовать для своих файлов имена встроенных модулей. ***

***❗❗❗ PEP-8! При импорте нескольких модулей каждый указывается с новой  строки. ***

```python
import sys  import random
```

# Использование from и as
([6](zotero://open-pdf/library/items/NW29V7HX?page=6&annotation=UN8LV5SF))

```python
from sys import builtin_module_names, path
```

***❗❗❗ PEP-8! Конструкция from import допускает перечисление импортируемых  имён объектов через запятую в одной строке. После from всегда указывается  один модуль. ***

> можно создавать псевдонимы для объектов через  зарезервированное слово as. При этом доступ к объекту будет возможен только  через псевдоним.
([7](zotero://open-pdf/library/items/NW29V7HX?page=7&annotation=QY7UC69D))

```python
import random as rnd  from sys import builtin_module_names as bmn, path as p  print(bmn)  print(*p, sep=&#39;&#92;n&#39;)  print(rnd.randint(1, 6))  print(path) # NameError: name &#39;path&#39; is not defined  print(sys.path) # NameError: name &#39;sys&#39; is not defined
```

***❗❗❗ Не стоит давать переменным короткие понятные лишь вам  имена. Код должен легко читаться другими разработчиками. Исключения общепризнанные сокращения, например import numpy as np. ***

> Плохой import * (импорт звёздочка)
([7](zotero://open-pdf/library/items/NW29V7HX?page=7&annotation=2PQVI27F))

	Подобная запись импортирует из модуля все глобальные объекты за исключением  тех, чьи имена начинаются с символа подчёркивания.

***❗❗❗ Если название объекта (переменной, функции и т.п.)  начинается с символа подчёркивания, объект становится защищённым. Если  имя начинается с двух подчёркиваний, объект становится приватным. Объекты  без подчёркивания в начале имени — публичные. ***

	Использование стиля from module import * зачастую приводит  к неожиданным результатам, затрудняет отладку кода и мешает верному  пониманию работы программы. Использовать подобный приём стоит в редких  особенных случаях. Кроме того импорт всех доступных объектов может  значительно замедлить работы программы, если таких объектов очень много.

# Переменная __all__
([9](zotero://open-pdf/library/items/NW29V7HX?page=9&annotation=UE5P45IM))

	При необходимости разработчик модуля может явно указать какие объекты нужно  импортировать при использовании стиля from module import *. Для этого  используется магическая переменная __all__

```python
from random import randint  9 __all__ = [&#39;func&#39;, &#39;_secret&#39;]  SIZE = 100  _secret = &#39;qwerty&#39;  __top_secret = &#39;1q2w3e4r5t6y&#39;  def func(a: int, b: int) -&gt; str:  z = f&#39;В диапазоне от {a} до {b} получили {randint(a, b)}&#39;  return z  result = func(1, 6)
```

	Переменной __all__ присваивается список имён объектов, заключённых в кавычки,  т.е. str типа. В основной модуль попадут только указанные в списке имена,  независимо от того являются они публичными, защищёнными или приватными. При  этом объект должен быть глобальным. Если указать в списке имя локального  объекта, например переменную z — локальную переменную функции func, получим  ошибку.

	Список __all__ в приведённом примере используется для формирования списка  импортируемых объектов модуля. Кроме этого __all__ применяется для импорта  модулей из пакета.

# __name__ == &#39;__main__&#39;
([13](zotero://open-pdf/library/items/NW29V7HX?page=13&annotation=KWCQ4DSD))

	команда import запускает импортируемый  модуль.

```python
...  if __name__ == &#39;__main__&#39;:  print(f&#39;{add(2, 4) = }&#39;)  ...
```

	Магическая переменная __name__ содержит полное имя модуля.

	Когда мы запускаем сам модуль base_math, в переменную __name__ попадает имя  “__main__”. Оно указывает, что файл запущен, а не импортирован.

	Когда модуль импортируется в другой файл, в переменную __name__ попадает его  имя, название файла без расширения.

	логическая проверка if __name__ == &#39;__main__&#39;: возвращает ложь и дальнейший  код пропускается.

# Создание пакетов и их импорт
([15](zotero://open-pdf/library/items/NW29V7HX?page=15&annotation=V2I6B8ID))

	В Python любая директория с файлом __init__.py автоматически становится  пакетом. При этом полезный функционал содержится в других питоновских файлах,  а не в “инит”.

	В  Python любой пакет является одновременно и модулем. Это означает, что пакет  можно импортировать в проект как и модуль. Так же это означает, что пакет может  хранить в себе другие пакеты — директории с “инит” файлом.

	Внутри файл __init__.py можно прописать код, который будет выполняться при  импорте пакета. Кроме того, в файл можно добавить переменную __all__ с именами  всех модулей пакета.

```python
__all__ = [&#39;base_math&#39;, &#39;advanced_math&#39;]
```

***❗❗❗ При добавлении или удалении модулей в пакет важно вносить  изменения в список __all__ для корректной работы импорта. ***

## особенности импорта внутри подпакетов.
([18](zotero://open-pdf/library/items/NW29V7HX?page=18&annotation=IXLBPFSV))

```python
from . import other_module
```

	если модулю надо выйти из своего пакета в пакет верхнего уровня, используют  вторую точку

```python
from .. import other_module
```

```python
from ..other_package import other_module
```

***❗❗❗ Модуль, который является основным в вашем проекте должен  использовать только абсолютные имена пакетов и модулей. Связано это с тем,  что у запускаемого модуля в переменной __name__ хранится значение  “__main__”, а не имя модуля. ***
