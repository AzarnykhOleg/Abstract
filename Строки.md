# Способы записи строк
([20](zotero://open-pdf/library/items/J46R29EE?page=20&annotation=Z4LXH72X))

	Python допускает одинарные или двойные кавычки для записи строки.

```python
txt = &#39;Книга называется &quot;Война и мир&quot;.&#39;
```

	Тройный двойные кавычки позволяют писать текст в несколько строк. Если такой текст не  присвоить переменной, он превращается в многострочный комментарий.

```python
class str(object):  &quot;&quot;&quot;  20 str(object=&#39;&#39;) -&gt; str  str(bytes_or_buffer[, encoding[, errors]]) -&gt; str  ...  &quot;&quot;&quot;
```

	Ещё один способ записать строки, перечислить их последовательно друг за другом.

```python
text = &#39;Привет.&#39; &#39;Как ты, друг?&#39; &#39;Рад тебя видеть.&#39;  print(text)
```

```python
very_long_text = &#39;Lorem ipsum dolor sit amet, consectetur  adipisicing elit. A ab alias animi assumenda at aut &#39; &#92;  &#39;commodi, consequatur cumque ea harum, hic id  illum ipsam itaque laboriosam magnam minus nam nulla &#39; &#92;  &#39;numquam obcaecati officia officiis porro  possimus praesentium quaerat temporibus ullam veniam? &#39;
```

	PEP8 рекомендует не писать более 120 символов в одной строке.  Обратный слеш “&#92;” позволяет писать продолжение с новой строки.

# Конкатенация строк
([21](zotero://open-pdf/library/items/J46R29EE?page=21&annotation=TFJ5NPEU))

```python
LIMIT = 120  ATTENTION = &#39;Внимание!&#39;  21 name = input(&#39;Твоё имя? &#39;)  age = int(input(&#39;Твой возраст? &#39;))  text = ATTENTION + &#39; Хоть тебе и осталось &#39; + str(100 - age) +&#92;  &quot; до ста лет, но длинна строки не должна превышать &quot; +  str(LIMIT) + &#39; символов.&#39;  print(text)
```

	● Все элементы должны быть строками. Иначе получим ошибку вида: TypeError:  can only concatenate str (not &quot;int&quot;) to str Именно для этого мы обернули  переменные с числами, такие как LIMIT, функцией str()  ● Если в переменной хранится строковое значение, оборачивать её в str() не  нужно. Получится масло масляное.  ● Складывать можно строки в одинарных, двойных и даже в трех двойных  кавычках. Но лучше выбрать единый стиль записи строк и придерживаться  его во всём проекте.  ● Если код не влазит в 120 символов строки, не стесняемся использовать  обратный слеш для его разделения на несколько строк.

> Конкатенация строк затратна по памяти и по времени.

# Размер строки в памяти
([22](zotero://open-pdf/library/items/J46R29EE?page=22&annotation=QA98KVTV))

```python
empty_str = &#39;&#39;  en_str = &#39;Text&#39;  ru_str = &#39;Текст&#39;  unicode_str = &#39;😀😍😉🙃&#39;  print(empty_str.__sizeof__())  print(en_str.__sizeof__())  print(ru_str.__sizeof__())  print(unicode_str.__sizeof__())
```

	метод __sizeof__(). Он работает  аналогично sys.getsizeof и возвращает количество байт занятых объектом.
